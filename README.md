## Fire Rescue Intelligent Communication Command System (SFRCS)

> **商用名称**：火眼搜援一体化系统  
> **当前版本**：V2.0  
> **文档状态**：产品规格细化中 · 原型研发准备阶段

---

## 📌 项目简介

本系统以**现有自组网通信设备为核心底座**（手持台 / 背负台 / 车载台 / 指挥台），通过叠加 AI 图像识别、语音事件结构化、多源感知融合（热成像 + 毫米波 + GIS）及任务闭环管理能力，将消防搜援现场的"感知碎片"转化为"可指挥的结构化信息流"。

### 核心价值

| 痛点 | 解决方式 |
|-----|---------|
| 视频回来了，但看不过来 | AI 自动识别重点目标，主动推送告警 |
| 发现了人，但上报靠喊话 | 语音事件结构化，自动生成事件卡 |
| 不知道哪里搜过、哪里没搜 | 搜救热力图 + 已搜/未搜网格 |
| 队员状态不透明、异常发现晚 | 腕表持续采集 + 异常自动置顶告警 |
| 主链路断了什么都不知道 | 低轨窄带保底，关键状态不断链 |
| 任务下发后无法追踪 | 任务闭环 + 腕屏回执 + 超时提醒 |

## 👥 目标用户与终端

| 角色 | 终端 | 核心诉求 |
|-----|-----|---------|
| 一线消防员（FF） | 腕屏 + 腕表 + 耳机 | 接令快、上报快、知道下一步 |

| 总指挥（CC） | 笔记本 | 战略决策 + 多部门协同 + 汇报留痕 |
| 通信保障（CT） | 笔记本链路管理页 | 保障链路稳定 |

## 📱 三端产品体系

### PC 端（笔记本指挥台）
- 综合态势总控页（多视频 + GIS + 事件任务三屏融合）
- AI 多源感知融合指挥页（AI识别第一亮点页）
- 搜救热力图页 / 人员安全看板页 / 任务闭环管理页
- 链路状态页 / 视频会商页 / 复盘训练页

### PAD / 手机端
- 班组长视图（GIS + 快速上报 + 热成像 + 传感器数据）
- 一线消防员视图（极简快操作：快速上报 + SOS + 任务接收）
- 个人作业视图（当前任务 + 生命体征 + 建筑平面图 + 语音指令）

### 腕屏端
- 完整模式（任务 + 距离 + 方位 + 生命体征 + 低帧视频）
- 极简模式（距离 + 方位 + 心率，一眼可读）

## 🎯 典型应用场景

1. 🏢 **城市建筑火灾内攻搜救** — 楼层平面图 + AI识别 + Mayday协同

4. 🌊 **洪涝孤岛转移救援** — 动态路径 + 多艇组分区调度
5. ⚗️ **危化事故现场侦察封控** — 暴露时长统计 + AI边界生成

## 🎨 设计语言

**工业功能主义（Industrial Functionalism）**

- 核心色系：深午夜黑 `#0F131D` + 救援橙 `#FF6B00` + 荧光绿 `#00E639`
- 字体：Inter（正文）+ Space Grotesk（数字/等宽数据）
- 原则：高对比度 · 信息密度分级 · 一线少操作强提醒 · 指挥员看全局抓重点
