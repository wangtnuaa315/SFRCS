## 一、产品定位与核心价值主张

### 1.1 一句话定位

> **以现有自组网通信设备为核心，通过 AI 图像识别、语音事件化、多源感知融合与任务闭环能力，将消防搜援现场的「感知碎片」转化为「可指挥的结构化信息流」，并以战术地图为核心大脑呈现。**

### 1.2 解决的本质问题

| 现有痛点 | 本系统解决方式 |
|---------|--------------|
| 视频回来了，但看不过来 | 视频降级为辅助验证手段，**AI 识别目标直接在地图打点**，主动推送告警。 |
| 发现了人，但上报靠喊话 | 语音事件结构化，自动生成事件卡。 |
| 不知道哪里搜过、哪里没搜 | 以地图为中心，叠加**搜救网格覆盖层**与轨迹热力图。 |
| 队员状态不透明、异常发现晚 | 左侧传感器矩阵 + 右侧人员生命体征卡片实时悬浮展现。 |
| 主链路断了什么都不知道 | 低轨窄带保底，关键状态（心率/坐标/SOS）不断链。 |
| 任务下发后无法追踪 | 任务闭环 + 腕屏回执 + 右侧 HUD 任务进度条实时追踪。 |

### 1.3 核心差异化（对外宣传口径）

1. **地图指挥大脑**：抛弃传统安防九宫格监控，视频只作为地图事件的补充，最大化全局战术视野。
2. **AI 识别驱动**：视频不再靠人眼盯，AI 识别人员、火点自动在地图生成任务坐标点。
3. **沉浸式 HUD 体验**：采用粗野主义与战术玻璃态（Editorial Brutalism），界面不干扰战场地图，科幻且高效。
4. **多源感知融合**：视频 + 热成像 + 毫米波 + 语音传感器流 + GIS 叠加融合。
5. **断网不断报**：Mesh 主链路 + 低轨窄带双保险机制，守护一线底线生命安全。

#### 4.1.2 各功能分区详细说明

**1. 顶栏 (Top Status Bar) - 态势仪表盘**
- 完全剥离页面导航功能，仅作为整个战场的全景数据概览区域。
- 只有数据读数：`系统身份` | `行动时长` | `覆盖率` | `链路状况` | `告警计数` | `实时时间`。
- 当出现 P0 告警时，整个顶栏告警区会散发红色脉冲泛光 (`box-shadow`)。

**2. 左侧边栏 (SideNav) - 唯一路由中枢**
- 只包含图标与双字标题的极窄导航区（宽度 72px）。
- 点击图标后，右侧大区域及相关面板联动切换。
- 最底部固定设置红色 SOS 一键广播按钮（长按 1.5 秒以上触发全频呼救防误触）。

**3. GIS 战术地图 (核心舞台)**
- 全屏沉浸式。所有操作（指派人员、查看位置、AI告警确认）都在地图上完成。
- 提供左下角悬浮楼层选择器 (B1/1F/2F/3F) 切片显示对应楼层的队员与搜救网格。

**4. 左侧 HUD 浮层 - 个体感知区**
- **视频区**：不再一味平铺九宫格。默认1路主看（带AI识别框），通过下方「卡片拨片」横向划动选择其他摄像头。AI告警命中哪个摄像头，主画面自动切换。
- **传感器矩阵 (Sensor Matrix)**：与当前视频或地图聚焦的队员联动绑定。直观打出该队员的心率进度条、血氧、气瓶、温度，以及具体的设备坐标和弱网状态标识。如果参数超标（如 BPM>150），出现「CRIT」标签。

**5. 右侧 HUD 浮层 - 战术指挥区**
- 三分天下布局（1:1.5:1.2分配）：
  - **右上 告警流 (Alert List)**：根据红、橙、黄分类显示所有自动化事件（SOS优先置顶）。点击「查看」会促使地图拉近。
  - **右中 任务流 (Task List)**：显示执行中、排队中、超时的任务；支持直接右键改派、或者拉起创建抽屉（联动地图打点下发任务）。
  - **右下 队员墙 (Personnel)**：极简的兵力清单。展示人员状态色块。如果是弱队标识，点击可直接发送语音指令或强行唤出其主视角视频。

#### 4.1.3 子路由独立页面定义

当通过 SideNav 切换时，主视图区的特殊模式：
- **AI 感知页**：地图退居右侧微型窗，视频与毫米波融合识别面板最大化呈现。
- **热力搜索页**：聚焦显示网格数据、已搜区域统计与历史行动轨迹动画回查。
- **通信链路页**：地图渲染为网络拓扑图（呈现树状 Mesh 基站-中继-终端 结构与在线干线）。
- **态势复盘页**：底部拉起多轨非线性时间线（音轨、视频轨、事件轨、地图回放轨）。

### 4.2 PAD 端与手机端（微缩现场指挥）

> PAD 端保持轻量化特征，重点在于**触控友好**与**班组隔离**。

**一线消防员视图 (FF)**
- “大字报”式展示当前排给我的任务；
- 底部三个核心巨大按钮：「语音对讲」「发送定位」「🆘」。
- 支持完全断网情况下的最后缓存数据查看。

**班组长视图 (TL)**
- 重点只展示本机小队（约 3-5 人）的摘要。
- 允许连接外接热成像仪（将 PAD 作为热成像主力大屏）。
- 快速创建与转发任务操作闭环。

---

### 4.3 腕屏端（手套盲操终端）

- **屏幕大小约束**：界面不超过 2 种颜色（常态白/绿，告警全红），内容限制在 3 行内。
- **状态流转机制**：
  - 常态下显示：`搜索 2区 | 剩余40min | 信号📶`
  - 收到任务或消息：全屏弹窗，震动提示。短按侧键一键「收到(ACK)」。
  - 告警发生（周边 SOS）：强烈组合震动，全屏幕闪烁红色。

## 五、典型场景完整用户旅程

*(融合 V3 HUD 设计风格演示)*

### 5.1 场景：厂房内攻搜索（含 AI 发现目标联动机制）

1. **部署发起**：指挥员 (IC) 在笔记本端左侧 SideNav 选「态势」，右中新建任务并指派。班长 TL 在 PAD 点击分配，突击队员腕屏震动确认。此时系统顶部 HUD 显示 `行动时长 00:00`，地图覆盖网格初始化。
2. **AI 发挥价值**：突击队员 FF-02 进入现场，摄像头捕捉到烟雾中模糊人形。AI边缘盒子产生推断，通过 Mesh 返回指令。
3. **指挥台反馈**：
   - 笔记本界面右上 `告警列表` 弹出紫色项：`AI 疑似被困者 (置信度 78%)`。
   - 地图中央弹射出对应的紫色脉冲 PIN 点。
   - 左侧 `视频轮切区` 自动切至 FF-02，画面中套用紫色边框和识别内容。
4. **决策循环**：指挥员点击告警项选择「生成任务」，侧滑抽屉自动带入上述坐标和图像快照，选择周围空闲的 FF-03 进行增援。
5. **设备掉卡与保底**：FF-03 进入信号黑洞。左侧 `传感器矩阵` 中 `链路稳定性` 掉到红色 `Narrowband` 状态。此时视频丢失，但因设备切至窄带卫星链路，其心跳与经纬度持续刷新，右下方人员墙不会报错「离失」。

## 六、核心能力模块技术规格

### 6.1 视觉AI与毫米波雷达分析标准

**AI 视频算子类别：**
- 被困人员检测 (倒地、半蹲)；高置信度 (≥70%)。
- 明火及热源扩散边界检测。
- 危险标志物或结构破损。

**毫米波与多源融合表现（V3 特性）：**
> 毫米波雷达不占据视频宫格！
- 毫米波捕捉微小人体呼吸频率（0.2-0.5Hz），在**地图层**输出为一个呈手电筒状的紫色半透明扇形扇区。
- 如果该扇区与热成像产生的红点、AI视频捕捉的框**相互重叠**，则引擎内部将「置信度」联合提升至 ≥90%，并将告警直接定性为「高可信目标」，触发红灯警报。

### 6.2 语音事件化解析引擎 (ASR + NLP)

- **UI 呈现位置**：常驻于系统底部的 `BottomBar` 黑色滚动带中。
- **工作机制**：现场所有通信语音统一经过 NLP 后端，一旦识别到诸如「发现伤员」「撤退」「没气了」等词包：
  - 即刻在右侧事件轴中打标签。
  - 生成一个「草稿任务」在中间等待指挥员右键指派。
  - 高亮截取带时间戳的语音音频，归档至「复盘」时间轴内。

## 七、数据流与联动逻辑

### 7.1 全局状态树联动 (基于 Pinia)

在 V3 的无刷新多浮层架构下，各组件必须依靠同一个状态机紧密联结：
- `focusedPersonnelId` 改变：导致【左下视频自动切过去】+【左下传感器读数更新】+【右下人员列表高亮红框】+【中央大地图自动飞到 (flyTo) 其坐标并闪烁】。
- `selectedAlertId` 改变：导致【地图平移定位】+【详情抽屉展开】+【对应视频通道激活】。
- `activeFloor` 改变：仅显示特定楼层下的网格、热力图、与相关楼层队员（非目标楼层人员透明度降到 20%）。

## 八、页面交互与视觉设计指南

### 8.1 视觉基调 (Editorial Brutalism)

本次重构抛弃大屏常用的软圆角玻璃态或蓝光科幻风，转向**高对比、实用主义、紧密感**的军事化特种 UI 路线：

1. **零圆角规范 (Zero-Radius)**：所有卡片、边框、进度条必须呈现锐利的直角边缘。
2. **战术玻璃态**：背景必须是极深的 `void黑` (如 `#030509` )，浮层采用纯黑半透明加极强的背景糊化（`backdrop-blur(12px)`）。
3. **幽灵边框 (Ghost Borders)**：采用 1px 厚度的高对比度（多为刺眼橙色或深空灰）边框线绘制容器。
4. **无衬线字体**：数字和英文硬性使用 `JetBrains Mono` 或 `Space Grotesk`，强调工业感精度。
5. **分段式进度条 (Segmented)**：代替原有的平滑连续加载，所有能量、血氧条表现为分段切割状。

## 九、功能优先级与MVP建设路径

### 第一阶段（1-2 个月）—— “点亮战场大脑” MVP
- 完成基于地图全场景 + HUD 左/中/右重构的前端主框架。
- 对接地图服务器，完成单兵 GPS 和心拍传感数据的 WebSocket 接入与面板渲染。
- 跑通文本告警流、简易工单分配。

### 第二阶段（3 个月）—— “融合多源视野”
- WebRTC 无延迟视频推流接通，视频拨片轮切。
- 集成 AI 盒子回接能力，把紫色的 AI 预测点、高管识别框成功绘制上去。
- 毫米波雷达、通信拓扑（链路页）联调。

### 第三阶段（5-6 个月）—— “数字化闭环”
- ASR 语音到文本转写系统集成在底部滚动带。
- 智能生成作战指令。
- NLE 复合时间轨（复盘界面）重现开发。
- PAD 和腕带边缘终端的系统研发与弱网脱网重连测试。

## 十、量化验收指标

1. **地图操控流度**：在加载全国全彩卫星底图 + 标记数 > 100 情况下，地图拖拽帧率 `> 50fps`。
2. **事件联动延时**：系统接收到一线传感器超标数据（通过 WebSocket）到右侧告警并使 UI 发红的时间差 `< 300ms`。
3. **视频切换速度**：在左侧 HUD 操作栏点击另一个人员摄像头，画面加载完成且 OSD 标识贴上耗时 `< 800ms`。
4. **组件盲操适配度**：侧边栏、HUD 开关全键盘映射可行（按 `1-7` 跳栏，空格控制视频），脱离鼠标可完成基本监控。
