# 消防搜援智能通信指挥系统 · 项目目录架构规划

> 基于：产品规格细化文档 V1.0 + UI视觉设计规范 全景战术版 V2.0 + 项目问答记录 Q01-Q11

---

## 一、技术栈（已确认）

| 层次 | 选型 | 说明 |
|-----|-----|------|
| **前端框架** | Vue 3 + Vite | 组件化、响应式 |
| **状态管理** | Pinia | Vue 3 标配 |
| **GIS 地图** | MapLibre GL JS | 开源、WebGL、原型/生产同一套代码 |
| **底图** | 原型 OSM 在线 → 生产 TileServer GL 离线 | **仅改 `.env` 一行 `VITE_TILE_URL`** |
| **样式** | Vanilla CSS + CSS Variables | OLED 战术色彩精确控制 |
| **字体** | JetBrains Mono（数值）+ Inter（文字） | 双界字体规范 |
| **AI 叠加** | Canvas 2D API（独立图层） | 不污染地图渲染 |
| **实时通信** | WebSocket | 心率/位置/AI识别推送 |
| **Mock 服务** | Express + ws | 演示开箱即用 |

## 四、关键约束

| 约束 | 规则 |
|-----|------|
| **同代码换配置** | 地图/API/WS URL 全在 `.env`，原型→生产零代码改动 |
| **Feature Flag** | 所有 Add-on 模块先查 `featureFlags.js`，未激活不渲染 |
| **demo 模式** | `VITE_DEMO_MODE=true` 自动注入 Mock，`DemoModeBadge` 显示角标 |
| **Canvas 隔离** | AI Canvas 与 GIS Canvas 独立，不互相触发重绘 |
| **颜色 Token 化** | 用 `var(--color-sos)` 引用，禁止硬编码颜色值 |
| **SOS 卡片规范** | 深底色 + 3px 橙边框，禁止大面积橙色块填充 |
