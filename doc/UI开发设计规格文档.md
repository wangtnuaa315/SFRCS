## 一、设计系统规范（Design System）

### 1.1 创意北极星：工业功能主义（Industrial Functionalism）

> **核心理念**：在高风险应急响应环境中，认知负荷是最大的敌人。
> 本设计系统摒弃"好莱坞 HUD"风格，不使用不必要的发光、科幻透明效果和装饰性杂乱元素。
> 转而采用植根于精确、可靠和清晰主义的**工业功能主义**风格。

### 1.2 颜色规范

#### 基础面板色（Foundation Colors）

| Token 名 | Hex 值 | 用途 |
|---------|--------|------|
| `background` | `#0F131D` | 页面最底层背景（基础空间感） |
| `surface-container-lowest` | `#0A0E18` | 最深底层，用于 GIS 地图背景层 |
| `surface-container-low` | `#1A1E2A` | 侧边栏、次级面板背景 |
| `surface-container` | `#20242F` | 标准卡片背景 |
| `surface-container-high` | `#262A35` | 浮动数据卡、「当前事件」模块 |
| `surface-container-highest` | `#2E3240` | 最高层浮动元素、输入框背景 |
| `on-surface` | `#E2E4EC` | 主要文字颜色 |
| `on-surface-variant` | `#E2BFB0` | 次要文字/标签颜色（偏暖） |

#### 战术指示色（Tactical Accent Colors）

| Token 名 | Hex 值 | 严格用途（禁止滥用） |
|---------|--------|------------------|
| `primary-container` / 救援橙 | `#FF6B00` | **仅用于**：SOS、活跃火点、紧急告警 |
| `tertiary` / 荧光绿 | `#00E639` | **仅用于**：安全、在线、已解决状态 |
| `error` | `#FFB4AB` | 错误状态文字 + 输入框左侧 2px 状态条 |
| `secondary-container` | `#444955` | 次要按钮、状态芯片（Chip）背景 |
| `surface-tint` | `#FFB693` | 浮动模态框的环境阴影着色 |

#### ⚠️ 颜色使用禁令

- **「救援橙」原则**：如果所有内容都用橙色，则橙色就失去了紧急含义。**仅在真正紧急时使用。**
- **禁止**：高亮度白色或亮灰边框（100% 白色或亮灰边框会造成视觉疲劳）
- **禁止**：分割线（1px 实线边框）用于区域分割，应通过背景色深浅变化形成边界

### 1.3 字体规范

#### 双字体策略

| 字体 | 用途 | 理由 |
|-----|-----|------|
| **Inter / PingFang SC** | Body、Title、Headline 所有正文角色 | 高可读性，汉字适配良好 |
| **Space Grotesk / Monospace** | label-md、label-sm（数字数据专用）| 数字对齐，「机器读取」感 |

#### 字体层级

| 层级 | Font | 典型用途 | 示例 |
|-----|-----|---------|------|
| Display | Inter Bold, 32px+ | 核心告警数量、关键任务标题 | `3 ACTIVE ALARMS` |
| Headline | Inter SemiBold, 20-28px | 区域标题、任务名称 | `3号楼火源侦察` |
| Body | Inter Regular, 14-16px | 卡片内容、事件描述 | 任务描述文字 |
| Label-md | Space Grotesk, 12-13px | 单位ID、状态标签 | `UNIT-07` / `10:42` |
| Label-sm | Space Grotesk, 10-11px | GPS坐标、传感器数值 | `22.5431°N, 114.0579°E` |

#### 数字数据必须使用等宽字体

所有数值型数据（坐标、时间戳、心率、温度、距离、百分比等）**必须**使用等宽字体（Space Grotesk 或 Monospace），确保对齐和「仪表盘」质感。

### 1.4 间距与圆角规范

| Token | 值 | 用途 |
|------|---|-----|
| `spacing-xs` | 4px | 组件内紧密元素 |
| `spacing-sm` | 8px | 列表行间距、组件内边距 |
| `spacing-md` | 16px | 卡片内边距、区域间距 |
| `spacing-lg` | 24px | 主要区域间距 |
| `spacing-xl` | 32px | 页面级间距 |
| `roundedness-sm` | 2px (0.125rem) | 大多数组件（保持「工业」感） |
| `roundedness-md` | 6px (0.375rem) | 卡片、面板 |
| `roundedness-lg` | 8px | 最大值，超过此值过于「消费品化」 |

### 1.5 层深（Elevation）规范

**禁止传统投影（box-shadow），使用「环境色调深度」替代：**

| 层级 | 背景色 | 适用场景 |
|-----|-------|---------|
| 底层 | `surface-container-lowest` (#0A0E18) | GIS 地图画布 |
| 侧边栏 | `surface-container-low` | 导航栏、侧面板 |
| 卡片 | `surface-container-high` | 数据卡片、「当前事件」模块 |
| 浮层 | `surface-container-highest` + backdrop-blur(20px) opacity 85% | 浮动地图控件、模态框 |

**浮动模态框**：使用 `surface-tint` (#FFB693) 着色的 4% 不透明度漫射阴影，营造物理屏幕层叠感。

**高密度视频宫格**（色调变化不够时的备选）：使用 `outline-variant` 颜色 15% 不透明度边框（作为边界暗示，而非硬性分割线）。

**工业玻璃效果**（地图控件浮层）：`surface-container` 颜色 85% 不透明度 + `backdrop-filter: blur(20px)`

### 1.6 组件规范

#### 按钮

| 类型 | 背景 | 文字 | 圆角 | 用途 |
|-----|-----|-----|-----|-----|
| **Primary / SOS** | `#FF6B00` | `on-primary` 白色 | 4px | SOS、紧急操作、核心行动 |
| **Success** | 深绿 `#1A4D2E` | `#00E639` | 4px | 任务完成、已解决状态 |
| **Secondary** | `#444955` | `on-surface` | 4px | 系统操作，融入背景直到 hover |

#### 数据卡片

- 背景：`surface-container-high`
- 无分割线（禁用），用 8px 垂直间距或交替行背景区分
- 状态指示：使用左侧 3px 竖条（颜色对应状态：橙/绿/红）

#### 遥测 Chip（状态标签）

- 背景：`secondary-container` (#444955)
- 字体：`label-sm`（Space Grotesk）
- 用途：单位ID、车辆状态、传感器数据

#### GIS 地图图标

- 线条式图标，2px 描边宽度
- 活跃事件：`#FF6B00`
- 人员位置：`#C2C6D5`
- 已完成任务：`#00E639`

  ```
  火源点 (FIRE) 99%        ← 红色矩形框 + 红底白字标签
  ```
  - 框线宽度：2px
  - 置信度数字：等宽字体，加粗
- **底部小视频宫格**：水平排列 4 格，显示不同视角/目标快照，带 `TARGET: XX` 标签

#### 右侧识别结果列表

- **「4 新发现」** 蓝色数量徽章（右上角）
- **结果卡片**（垂直列表）：
  ```
  [15分钟时间戳（等宽）]
  事件标题（Headline）
  坐标（label-sm，等宽）
  [高优先级] [同步地图...] ← 两个操作芯片
  ```
  - 不同事件类型用左边框颜色区分（人员=橙，火点=红，障碍=灰）

- **AI辅助决策建议**：
  - 标题带 `>` 引导箭头图标
  - 建议文字：14px，前方带橙色圆点
  - 每条建议独立行，简短直接

## 三、跨端统一交互规范

### 3.1 全局颜色语义（必须跨端一致）

| 颜色 | 含义 | 所有端均适用 |
|-----|-----|------------|
| `#FF6B00` 救援橙 | 紧急/SOS/活跃火点/CTA | ✅ |
| `#00E639` 荧光绿 | 安全/在线/完成 | ✅ |
| `#FFB4AB` 错误红（柔） | 错误/危险参数 | ✅ |
| `#C2C6D5` 冷灰 | 普通人员标记/次要信息 | ✅ |
| 暗红底+橙字 | 告警背景卡 | ✅ |

### 3.2 响应式适配层级

| 终端 | 分辨率范围 | 布局 |
|-----|---------|-----|
| PC端笔记本 | 1920×1080 ~ 2560×1440 | 三列布局，信息高密度 |
| PAD（横屏） | 1280×800 ~ 1366×768 | 两列布局，班组长视图 |
| 手机（竖屏） | 390×844 ~ 428×926 | 单列布局，极简快操作 |
| 腕屏 | ~390×390 | 极简单信息 + 三键 |

### 3.3 动效规范

| 类型 | 效果 | 时长 | 适用场景 |
|-----|-----|-----|---------|
| 告警出现 | fade-in + 轻微 scale(1.02) | 200ms | P0告警弹出 |
| SOS 按钮激活 | 橙色脉冲光环扩散（只做一次） | 600ms | SOS触发确认 |
| 卡片切换 | slide-in from right | 150ms | 任务队列切换 |
| 正常数据刷新 | 无动效（直接更新数据） | — | 心率、位置等 |
| 视频识别框 | 直接绘制，无过渡动效 | — | AI识别框出现 |
| 状态变化 | 颜色 cross-fade | 300ms | 任务状态变更 |

**禁止动效**：
- 不使用发光（glow）效果
- 不使用旋转/弹跳类动效（过于游戏化）
- 不使用持续循环动效（容易分散注意力）

## 四、开发技术建议

### 4.1 前端技术栈建议

| 层面 | 建议方案 | 理由 |
|-----|---------|-----|
| **PC端框架** | Vue 3 + Vite / React | 组件化复用，数据驱动 |
| **地图引擎** | Mapbox GL JS / Leaflet | 暗色风格支持，GeoJSON渲染 |
| **视频播放** | WebRTC + HLS | 实时流媒体，延迟低 |
| **AI识别框叠加** | Canvas + requestAnimationFrame | 高性能绘制，不影响视频帧 |
| **图表/波形** | ECharts / Chart.js | 毫米波波形，热力图叠加 |
| **移动端/PAD** | Vue 3 (响应式) / React Native Web | 与PC端共用组件库 |
| **腕屏** | 轻量 HTML/CSS/JS | 保持极简，无重框架 |
| **状态管理** | Pinia / Redux Toolkit | 跨组件实时数据同步 |
| **WebSocket** | 原生 WebSocket / Socket.io | 实时数据推送 |

### 4.2 性能关键要求

| 场景 | 要求 |
|-----|-----|
| AI识别框渲染 | 不阻塞视频播放，使用 `<canvas>` 叠加而非 DOM 操作 |
| GIS地图重绘 | 使用 WebGL 渲染（Mapbox GL 原生支持），图层按需显示 |
| 多路视频 | 非激活视频格降帧（≤5fps），激活后全帧率 |
| 实时数据刷新 | 心率/位置 ≤3s，不使用全页面重渲染 |
| 离线缓存 | Service Worker 缓存地图瓦片、任务列表 |

### 4.3 组件优先级（开发顺序建议）

**Phase 1（基础组件库）**：
1. `DesignTokens.css` — 全部颜色/字体/间距变量
2. `TopBar` — 顶部导航栏
3. `DataCard` — 数据卡片基础组件
4. `StatusBadge` — 状态徽章（颜色+文字）
5. `AlertBanner` — 告警横幅
6. `ActionButton` — Primary/Secondary/SOS/Success 按钮

**Phase 2（功能组件）**：
7. `GISMap` — 地图组件（含图层控制）
8. `VideoCell` — 单路视频格（含AI识别框叠加）
9. `VideoGrid` — 多路视频宫格
10. `TaskCard` — 任务卡片（含状态条）
11. `TaskList` — 任务列表
12. `VitalSignCard` — 生命体征卡

**Phase 3（页面合成）**：
13. `CommandDashboard` — PC端综合指挥台
14. `AIRecognitionPage` — AI识别指挥页
15. `FieldCommanderPAD` — PAD班组长视图
16. `FirefighterMobile` — 手机一线视图
17. `PersonalTaskView` — 个人任务视图
18. `WristDisplay` — 腕屏完整版
19. `WristMinimal` — 腕屏极简版
