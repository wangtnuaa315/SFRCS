# 综合态势总控页 V2 布局改版方案
> 文件性质：改版设计提案，不修改原规格文档。
> 参考来源：`stitch-202604131605/` 设计方案 + 消防搜援智能通信指挥系统产品规格。
> 核心改版动因：原有方案将"主视频"置于屏幕中央约 40% 宽度，在多源设备接入场景下视频信息密度失衡，战术地图与传感器数据反而被压缩。

---

## 1. 改版设计哲学（Design North Star）

**从"以视频为中心的监控大屏"转变为"以战术地图为大脑的指挥作战界面"。**

参考设计的核心洞察：
> 视频不是系统的"主角"——战术地图才是。视频是对战场信息缺失的一种补充入口，而不应占据超过 25% 的屏幕面积。

具体转变：

| 维度 | V1 原版 | V2 改版 |
|------|---------|---------|
| 布局模型 | 横向三列等分（地图/视频/右侧面板） | **地图全屏背景** + 左/右浮层 HUD |
| 视频占比 | 主视频约 35% 宽度 | 视频缩小为左侧 HUD 内的**浮窗模块**（约 25% 屏宽 × 20% 屏高） |
| 信息密度 | 视频 + 地图竞争核心区 | 地图独占中央，各类数据以**玻璃态悬浮面板**叠加 |
| 多路视频 | 底部 4 格宫格横排 | **收进左侧视频模块**，支持横向滚动轮切 |
| 传感器数据 | 右侧人员卡片列表 | 左侧**传感器矩阵区**，紧凑型数据块，可折叠 |

## 3. 分区详细规格

### 3.1 顶部状态栏（Top Status Bar）

> ⚠️ **V2 关键修正：顶栏不再承担页面导航功能，Tab 全部去除。**
> 顶栏唯一职责是"实时态势数字仪表盘"——所有内容只读展示，无可点击的页面跳转操作。

顶栏三区从左到右：

**① 系统身份区（左对齐，固定）**
- `🔴 火眼搜援`（品牌标识，橙色，JetBrains Mono 粗体，带 Tactical Glow）
- `|` 分隔线
- `任务 · 3号楼内攻搜救`（任务名，Inter，可动态更新）

**② 实时态势指标区（居中，只读数字仪表盘）**

| 指标项 | 格式示例 | 颜色 |
|--------|---------|------|
| 任务时长 | `43:21` | `--color-accent-safe`（绿色） |
| 已搜覆盖 | `47%` | `--color-accent-safe`（绿色） |
| 主链路 | `MESH ● 在线` | 绿色实心圆点 |
| 副链路 | `SAT · 待机` | `--color-text-muted`（灰色） |

**③ 状态汇总区（右对齐）**
- 告警计数：`⚠ 3`（有 P0 时整区出现橙色脉冲发光，`box-shadow: 0 0 8px rgba(255,60,0,0.6)`）
- 当前时间：`14:23:45`（JetBrains Mono，每秒更新，只读）

### 3.2 左侧竖向导航菜单（Side Nav）— 唯一页面导航入口

> ✅ **V2 设计决策：左侧 SideNav 是系统中唯一的页面级路由入口，顶栏不再重复此功能。两者职责彻底分离。**

**宽度**：`72px`，图标 + 两字标签形态（垂直排布）。  
**背景**：`rgba(3, 5, 9, 0.96)` 不透明，与地图浮层严格视觉分离（不用玻璃态）。  
**左侧边框**：无；右侧有 `1px solid rgba(255,60,0,0.12)` 极细橙色分隔线。

### 3.3 GIS 战术地图（中央背景层）

**视觉主角，占屏全部中央空间（扣除左侧72px导航 + 左侧HUD浮层 + 右侧HUD浮层）。**

地图上叠加以下可见元素（均为绝对定位的 MapLibre GL 层）：
1. **队员位置标记**：绿色菱形 PIN + 队员 ID 标签 + 楼层标签
2. **AI 疑似目标点**：紫色脉冲圆圈 + 置信度百分比 + 简短标签（悬停展开详情）
3. **搜救网格覆盖层**：绿色半透明填充（已搜） / 暗灰（未搜）
4. **火点标注**：橙红色菱形 + 闪烁动效
5. **图例**（左下角，贴靠 HUD 浮层下方）：小型图例台，内含圆点颜色说明

**底部浮条（中央）：** 宽度约 400px，高度 56px，玻璃态背景，展示 3 项关键环境指标：
```
火情等级: CAT-03   |   已确认幸存: 2   |   风向 NW: 14m/s
```

## 5. 视觉设计系统（沿用 stitch 方案）

### 色彩直接沿用 tokens.css 中的既有变量

- `--color-accent-sos: #FF3C00`（SOS/火点/告警 P0）
- `--color-accent-safe: #00E676`（队员正常/在线/已搜区域）
- `--color-accent-ai: #9333EA`（AI 疑似目标/热力分析）
- `--color-accent-warning: #F59E0B`（告警 P1/心率偏高）
- `--color-surface: #030509`（背景 Void Black）
- `--color-border: #1E293B`（Ghost Border）

### 字体规范（扩展 stitch 方案）

- **数据标签**（坐标/时间戳/心率数值）：`JetBrains Mono`
- **模块标题/区域ID**：`Space Grotesk` Bold（已通过 Google Fonts CDN 引入）
- **中文说明/告警内容**：`Inter`

### 关键视觉规则

1. **Zero Radius**：所有容器、按钮、标签均为直角（`border-radius: 0`）
2. **Tactical Glow**：SOS 告警卡片使用 `box-shadow: 0 0 20px rgba(255,60,0,0.4)`
3. **Segmented Progress**：进度条使用块状分段（不使用平滑填充）
4. **Ghost Border**：容器边框使用 `1px solid rgba(255,60,0, 0.15)` 而非实线

## 7. 交互规格（Interaction Specifications）

> 本章为每个模块定义完整的用户操作 → 系统响应 → 数据联动链路。
> 格式：`[触发方式]` → `[即时视觉反馈]` → `[数据状态变化]` → `[跨模块联动]`

### 7.1 顶部导航 Tab

#### 7.1.1 切换页面 Tab

```
用户操作: 点击顶栏 Tab（如"AI感知"）
──────────────────────────────────────────
即时反馈:
  - 点击 Tab 底部出现 2px 橙色下划线（--color-accent-sos）
  - 其余 Tab 文字变为 --color-text-muted
  - 主内容区淡出（opacity 0→1，duration 150ms）→ 新页面内容淡入
状态变化:
  - router.push('/ai-recognition') 或通过 Pinia activeTab 切换视图
跨模块联动:
  - 左侧 SideNav 对应图标同步高亮（保持双导航一致性）
  - 若切换至"人员安全"Tab，右下人员摘要区展开为全屏人员看板
```

#### 7.1.2 告警计数角标

```
触发条件: 后端推送新 P0/P1 告警（WebSocket 消息 type: 'ALERT'）
──────────────────────────────────────────
即时反馈:
  - 告警数字 +1，数字以 scale(1.4)→scale(1) 弹跳动效提示
  - P0 级别：顶栏告警区域边框出现 0 0 8px rgba(255,60,0,0.6) 脉冲发光
  - 同时触发浏览器 Notification API（如已授权）
状态变化:
  - alerts 数组 unshift 新告警项
跨模块联动:
  - 右上告警列表自动展开并将新告警置顶
  - 若新告警含 videoChannelId → 左侧视频模块自动切换至对应摄像头
  - 若新告警含队员 ID → 右下人员摘要该行红框闪烁
```

### 7.9 全局联动状态机（Cross-Module State Machine）

以下是各模块状态的统一触发链，通过 **Pinia 状态树** 驱动：

```
Pinia 核心状态:
  focusedPersonnelId    → 驱动: 传感器矩阵 + 视频切换 + 地图 flyTo + 人员摘要高亮
  selectedMapTargetId   → 驱动: AI Popup + 视频切换 + AI建议栏
  activeVideoChannelId  → 驱动: 视频主窗 + 缩略图高亮
  activeFloor           → 驱动: 地图楼层层级 + 覆盖率统计
  alerts[]              → 驱动: 顶栏角标 + 告警列表 + 全局告警音
  tasks[]               → 驱动: 任务列表 + 超时监控定时器
```

```
联动示例：FF-02 SOS 告警触发后的完整连锁反应：

1. WebSocket 收到 { type: 'SOS', from: 'FF-02' }
2. alerts.unshift(新告警)
3. 顶栏告警角标 +1，弹跳动效
4. 告警音播放
5. 右上告警列表自动置顶展示 SOS 条目（红色闪烁）
6. focusedPersonnelId = 'FF-02'
   └── 左侧传感器矩阵切换至 FF-02 数据
   └── 视频切换至 CAM-02（FF-02摄像头）
   └── 地图 flyTo FF-02 坐标，PIN 闪烁
   └── 右下人员摘要 FF-02 行红框
7. 左侧 SideNav SOS 图标出现红点角标
```
