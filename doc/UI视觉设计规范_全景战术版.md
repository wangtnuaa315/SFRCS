# 消防搜援智能通信指挥系统 - UI视觉设计规范 (全景战术版)

> **文档版本**：V3.0 (适配全景态势页最新改版方案)
> **设计基调**：Editorial Brutalism (粗野编辑主义) + 战术玻璃态 (Tactical Glassmorphism)
> **适用终端**：PC 笔记本端综合态势总控页、大屏展示端

---

## 1. 核心设计哲学 (Design Philosophy)

本套 UI 体系彻底抛弃了传统的“后台管理系统分栏”与“安防九宫格监控”模式，转向**军事/特种装备级**的界面质感。追求信息密度、暗态高对比度与极致的冰冷专业感。

1. **地图即大脑 (Map as the Brain)**：GIS 战术地图全屏无缝铺满，彻底取消传统布局中的面板切割区。
2. **悬浮视界 (Floating HUDs)**：所有的监控视频、传感器数据、任务卡片、导航条均以“悬浮舱”的形式透明叠加于地图上方，打造科幻且具有纵深空间感的 HUD (平视显示器) 体验。
3. **拒绝光污染与圆角软化**：克制使用高饱和度亮色填充；严格实行“零圆角”纪律；告警与提示仅通过极细的线条和锐利的文字发光表现。

## 2. 空间布局与层级规范 (Spatial Layout System)

整个页面分为 **底层 (Layer 0)** 与 **HUD悬浮层 (Layer 1)**。

| 区域模块 | 布局定位约束 | 尺寸与表现规范 |
|---------|-------------|--------------|
| **GIS 战术图** | 主体底层 (`z-index: 10`) | `100vw × 100vh` 全覆盖，操作级底色压暗。 |
| **顶栏态势条** | T: 0, L: 0, 悬浮于顶 | 高度 `48px`，去导航化，纯数据仪表盘呈现，底色 `#000000`。 |
| **左侧导航轴** | T: 48px, L: 0 | 极窄设定（宽 `72px`），系统唯一路由入口。 |
| **中央底层浮条** | B: 36px+间隙, 水平居中 | 宽 `420px` 左右，高 `56px`。展示 3~4 项核心环境极简指标，战术玻璃态。 |
| **左侧感知区** | L: 72+16px, 紧靠左导航 | 宽约 `300px`，承接视频拨片轮切窗与传感器矩阵。 |
| **右侧指挥区** | R: 20px, 贴右 | 宽约 `320px`。高度实行 `Flex` 比例分配（告警 2 : 任务 1.5 : 人员 1.2）。 |
| **底部语音带** | B: 0, L: 72px | 高 `36px`，黑色底条。左侧系统健康区，中间跑马灯语音条。 |

*(注：HUD面板需在底部留出 `16px` 以上 padding，确保它们在视觉上如悬浮状“飘”在底部数据条上方。)*

## 3. UI 视觉表现纪律 (Strict Visual Rules)

### 3.1 Refined Radius (细腻倒角边缘)
- **规则**：告别极端的绝对锐角，针对主要的悬浮 HUD 面板（视频、传感器、数据列表面板）和底部环境信息条，统一采用 **`8px` 的克制圆角 (`border-radius: 8px`)**。
- **目的**：在保持军事级硬核科幻感的同时，稍微缓解原本完全直角的攻击性，使得漂浮叠层的 UI 更具现代工业设备的圆润精致感。内部次级元素（如小标签、内部细分列表）仍可保持无圆角或更小圆角，以形成对比。

### 3.2 Tactical Glassmorphism (战术玻璃态)
- **规则**：HUD 容器背景严禁使用实色，必须使用诸如 `rgba(16, 20, 25, 0.75)` 的深黑通透底色，并叠加 `backdrop-filter: blur(12px)`。
- **效果**：不仅能透视底层的地图网格，还能在滑动地图时看到背靠的颜色韵律，增强沉浸感。

### 3.3 Ghost Borders (幽灵极细边框)
- **规则**：不使用传统的深色加粗边框，大量采用 `1px solid rgba(255, 255, 255, 0.08)` 构建结构边界。
- **高亮态**：选用高饱和发光单体色，如 `1px solid rgba(255, 60, 0, 0.5)` 作为选中指示。

### 3.4 交互层穿透 (Pointer-Events)
- 为了保证全屏地图不受 HUD 层占位影响，`HUD包围盒` 必须设置 `pointer-events: none`（鼠标穿透）。
- 仅为实际面板实体（如 `.hud-module`）设置 `pointer-events: auto`，确保地图的拖拽区最大化。

## 4. 全景战术色彩与排版词典 (Color & Typography)

### 4.1 色彩变量系统 (Design Tokens)

以 CSS Variables 为准，禁止代码层面写死 hardcode 颜色：

| 变量定义 | 核心色值 | 战术语义与场景限制 |
|---------|---------|------------------|
| `--color-surface` | `#030509` (极黑) | 战术地图垫底色、独立纯黑条块 |
| `--color-border` | `#1E293B` | 透明面板间的切割线基础色 |
| `--color-accent-sos` | `#FF3C00` (霓虹橙) | **MAYDAY**！火点、P0级紧急告警框、异常心率（仅做极细线或单字发光） |
| `--color-accent-safe`| `#00E676` (雷达绿) | 正常心率、设备上线/健康态、已扫描覆盖网格、任务进度 |
| `--color-accent-ai` | `#9333EA` (雷达紫) | AI识别框、毫米波探测区域、热力分析脉冲波 |
| `--color-text-primary`| `#F8FAFC` (冰白) | 面板标题、核心数值展现 |
| `--color-text-muted` | `#64748B` (烟蓝) | 辅助单位(BPM, m/s)、次要时间戳记 |

### 4.2 字体排版系统

采用**双界字体策略**，区分阅读区域与战术读数区：

1. **战术读数 (Tactical Data)**
   - **字体**：`JetBrains Mono`（优先）或等宽英文字体。
   - **场景**：经纬度坐标、时间戳、心率血氧数字、风速、MAC/IP地址。
   - **排版纪律**：数值旁边必须带上干练的单位缩写小尾巴（如 `<span class="mono-normal">14m/s</span>`，且单位字号降低以凸显数字本身）。
2. **模块头与常规内容 (General Text)**
   - **字体**：`Space Grotesk` (粗体英文大标题) 及 `Inter` (中文主体)。
   - **说明**：清晰易读，适用于界面导航名、任务阐述文本、系统警告语。

## 5. UI 组件细节示例

### 5.1 SOS 按键与状态灯
摈弃大面积红/橙色填充（反光污染原则）。
正确做法：纯黑/透明底框 + 正中图标/文字显示橙色 `#FF3C00`。若需强调，可为其外层容器增加浅色幽灵发光圈 `box-shadow: 0 0 16px rgba(255, 60, 0, 0.3)`。

### 5.2 进度条呈现
严禁使用平滑渐变加载条。必须使用**Segmented (分段切割块状)**表达方式。例如展现 `65%` 执行进度，采用：`▓▓▓░░`，更符合恶劣通信环境下的容错仪表板感。

### 5.3 数据分栏线
数据项之间的分割采用 `.env-divider` 定位块，高度仅占父级 `40%` 左右，粗细 `1px`，用淡灰透明色切断两个数据组，保持高度视觉透气性。
